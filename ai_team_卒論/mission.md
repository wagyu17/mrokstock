# Mission

このAIチームの目的は、静圧空気スピンドルのエンドミル加工シミュレーション高度化に関する **卒業論文** と **学術誌投稿論文** を、Claude Code・Codex・Antigravity の 3AI 体制で完成させることである。

## 2つのミッション

1. **エンドミルたわみの可視化**: たわみ角を出力し、実際にたわむ様子を3Dシミュレーション or 画像として可視化する
2. **学術誌投稿論文の作成**: 嶋田論文（Proc IMechE Part C, 2022）をベースに、下八川→中村→丸岡の研究成果を追加した論文を執筆する（＝卒論と同内容、2027年2月末完成）

## 役割

- **Claude Code（PM・設計）**: 研究計画管理、Fortran修正のロジック設計、論文構成・ドラフト、ゼミ発表のレビュー、次アクション整理
- **Codex（実装）**: Fortranコード修正、Pythonスクリプト作成、HTML可視化ツール改修、バッチ計算ツール、データ整形・図表生成
- **Antigravity（リサーチ・QA）**: 先行研究調査、論文引用の確認、可視化ツールのブラウザ動作検証、実験データの整合性確認

## 基本サイクル

1. Claude Code が現在フェーズの作業計画を立てる
2. Antigravity が必要な調査・検証を実施する
3. Claude Code が調査結果を踏まえて Codex への実装指示を作成する
4. Codex が実装を行う
5. Claude Code が実装結果をレビューする
6. Antigravity が動作検証を行う
7. Claude Code が次タスクを決定する

## 可視化戦略

**方針**: `bearing_gap_simulator.html` を拡張し、解析結果の可視化機能を集約する。個別の画像・グラフではなく、インタラクティブな 3D ツール上で全てを表示する。

### 可視化要素の優先順位

| 優先度 | 要素 | 内容 | Fortran依存 | 論文での用途 |
|--------|------|------|-------------|-------------|
| **P1** | v1hy 実データ表示 | endmill_deflection.csv に y方向たわみ列を追加し、3D エンドミルを xy 両方向にたわませる | Phase 3 必須 | 第4章: xのみ vs xy の比較 |
| **P2** | 2D合成たわみ | `sqrt(v1h²+v1hy²)` の合成値と方向ベクトルを数値＋矢印で表示 | Phase 3 必須 | 第4章: 実切込み深さの改善実証 |
| **P3** | たわみ角 φx, φy | `atan(Δv1h/Δze)`, `atan(Δv1hy/Δze)` をリアルタイム数値＋矢印で表示 | Phase 3 必須 | 第4章: 切削面傾斜角度との比較 |
| **P4** | 実験値重畳表示 | 実験測定の振れ量データを同一 3D 空間に重ねて表示 | 不要 | 第4章: 数値 vs 実験の定量比較 |
| **P5** | 切削面プロファイル | たわみ角から予測される加工面傾斜を 2D 断面図で表示 | Phase 3 推奨 | 第4章4.6節: 可視化の有用性 |

### クリティカルパス

```
Phase 3 Fortran実装（v1hy + CSV出力拡張）
  → P1: HTML側で v1hy 読み込み・3D表示
  → P2: 合成たわみの計算・表示
  → P3: たわみ角の計算・表示
  → Phase 4: 64シナリオ計算
  → P4: 実験値重畳（Phase 4 と並行可）
  → P5: 切削面プロファイル
```
