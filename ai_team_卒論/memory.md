# Memory

卒論プロジェクトで継続的に覚えておくことを書く。

## 決定事項

- 研究テーマ: 静圧空気スピンドルを用いたエンドミル加工シミュレーションの高度化
- 前任者: 中村朋哉（修士）、指導教員: 宮武正明 教授（東京理科大学）
- 担当: 丸岡大也（学部4年、学籍番号 4523139）
- 卒論提出: 2027年1月、発表: 2027年2月
- Fortranコードのビルドには Intel oneAPI Fortran Compiler が必要

## 用語

- **v1h(qqq)**: エンドミル x 方向各断面のたわみ量 [m]
- **v1hy(qqq)**: エンドミル y 方向たわみ（Phase 3 で追加予定、未実装）
- **v2h(qqq)**: ねじれ角を考慮したたわみ
- **ddd**: たわみ収束判定係数（二分法）
- **d1 / d2**: D=1mm / D=2mm 工具径の区分
- **M1 / M2**: 実験装置のバージョン
- **kk**: エンドミル z 方向分割数（最大1000）
- **odi**: 出力間隔（ステップ数）
- **typeC**: 切削実験の工具タイプ

## 指導教員からの指摘

- コード全体を完全に理解する必要はない。入出力関係を把握し、卒論の目的に直結する部分（切削力・たわみ）を重点的に理解する
- 可視化ソフトは研究補助ツールとして位置づける（計算本体ではなく後処理）
- ゼミ発表では一度にすべてを見せず、「完成物1つ + 分かったこと + 次にやること」の形で段階的に
- 実装は進めてよいが、発表は自然な単位に分ける
- AIにコードを書かせること自体を成果とせず、目的・構造・使い方・限界を自分で説明できること

## 注意点

- Fortranコードは約10,000行の単一ファイル（2025.for）— 行番号が変わると他の参照がずれる
- 実験生データの .cag ファイルは各 ~570MB — Git管理外にすべき
- INPUTparameter.txt の行28（Ad）が空欄だとたわみ計算がスキップされる
- 可視化ツールの Y 方向たわみ表示は近似値（Fortran未実装のため JS 側で対称仮定で計算）
- `ADVANCE='NO'` は Intel Fortran Compiler 専用構文
