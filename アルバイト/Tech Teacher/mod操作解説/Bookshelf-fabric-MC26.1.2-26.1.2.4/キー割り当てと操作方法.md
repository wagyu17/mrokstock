# Bookshelf キー割り当てと操作方法

## MOD概要

Bookshelf は、多くのMODが共通処理を利用するためのライブラリMODです。単体で遊び方を大きく変えるMODではなく、他MODの前提として導入されます。

## 解析元

- jar: `Bookshelf-fabric-MC26.1.2-26.1.2.4.jar`
- modId: `bookshelf`
- loader: Fabric
- 主な解析対象: `fabric.mod.json`, `net/darkhax/bookshelf/**`

## キー割り当て

Bookshelf 専用の通常プレイ用キー割り当ては検出されませんでした。

## 独自コマンド

Bookshelf には、デバッグ・テスト・開発補助用のコマンドクラスが複数含まれています。通常プレイで必須のものではありません。

解析で確認できた主なコマンド要素は次の通りです。

| コマンド候補 | 内容 |
| --- | --- |
| `/bookshelf debug ...` | チェスト、ディスペンサー、壺、スポナー、ルートテーブルなどのデバッグ |
| `/bookshelf enchant <enchantment> <level>` | 手持ちアイテムなどへのエンチャント付与補助 |
| `/bookshelf rename <new_name>` | アイテム名変更の補助 |
| `/bookshelf hand ...` | 手持ちアイテムのID、NBT、コンポーネント、タグ確認 |
| `/bookshelf font ...` | フォント・メッセージ表示関連のテスト |
| `/bookshelf translate <key>` | 翻訳キー確認 |
| `/bookshelf structure ...` | 構造物関連の確認 |

クラス内で検出された文字列には `debug`, `chests`, `dispensers`, `gameplay`, `pots`, `spawners`, `enchant`, `rename`, `hand`, `translate`, `structure` などがあります。実際の利用可否は、権限・環境・Bookshelfを利用する側のMOD構成に依存します。

## 操作方法

### プレイヤーとして使う場合

Bookshelf は前提MODです。プレイヤーが直接操作するというより、Bookshelfを必要とするMODが内部で利用します。

1. Bookshelf を必要とするMODと一緒に導入します。
2. loader が一致しているか確認します。このjarは Fabric 版です。
3. Minecraftを起動し、依存MODが正常に読み込まれれば導入完了です。

### 開発・検証で使う場合

コマンド機能は、MODパック制作や検証時にアイテム・タグ・翻訳・ルートテーブルなどを確認する目的で使います。授業用の通常プレイでは説明優先度は低めです。

## 注意点

- このjarは Fabric 版です。NeoForge環境にそのまま入れても基本的には動作しません。
- Bookshelfを必要とするMODがなければ、単体で目に見える変化は少ないです。
- デバッグ系コマンドは権限が必要な場合があります。サーバーで使う場合は管理者権限を前提にしてください。
