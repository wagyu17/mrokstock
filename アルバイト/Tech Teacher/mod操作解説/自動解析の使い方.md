# MOD jar 自動解析の使い方

CurseForge などからダウンロードした jar を、同じ名前のフォルダに格納し、`キー割り当てと操作方法.md` の下書きを自動生成するための手順です。

## できること

`mod解析自動化.ps1` は、次の処理をまとめて行います。

1. jar ファイル名と同じフォルダを作成する
2. そのフォルダへ jar をコピー、または移動する
3. jar 内の `mods.toml`、`fabric.mod.json`、`mcmod.info` などを読む
4. lang ファイルからキー表示名を拾う
5. class ファイルからキー割り当て候補と独自コマンド候補を探す
6. `キー割り当てと操作方法.md` を生成する
7. 詳細確認用に `解析ログ.json` を保存する
8. Codex に追記を依頼するための `Codex追記依頼.md` を生成する

## 基本の使い方

PowerShell で `mod操作解説` フォルダを開いてから実行します。

```powershell
cd "C:\Users\tomot\Desktop\management\アルバイト\Tech Teacher\mod操作解説"
powershell -ExecutionPolicy Bypass -File .\mod解析自動化.ps1 -JarPath "$HOME\Downloads\ダウンロードしたmod.jar"
```

この場合、ダウンロード元の jar は残したまま、`mod操作解説` フォルダ内へコピーします。

## 最新のダウンロード jar を解析する

`Downloads` フォルダ内で一番新しい `.jar` を自動で選ぶ場合は、次のように実行します。

```powershell
powershell -ExecutionPolicy Bypass -File .\mod解析自動化.ps1 -LatestFromDownloads
```

## jar をコピーではなく移動する

ダウンロードフォルダから jar を移動して、作成したフォルダ内だけに置きたい場合は `-Move` を付けます。

```powershell
powershell -ExecutionPolicy Bypass -File .\mod解析自動化.ps1 -LatestFromDownloads -Move
```

## Markdown を作り直す

すでに `キー割り当てと操作方法.md` がある場合、上書きはしません。作り直したい場合だけ `-OverwriteMarkdown` を付けます。

```powershell
powershell -ExecutionPolicy Bypass -File .\mod解析自動化.ps1 -JarPath "$HOME\Downloads\ダウンロードしたmod.jar" -OverwriteMarkdown
```

## 生成されるファイル

例として `example-1.20.1.jar` を解析した場合は、次のようになります。

```text
mod操作解説/
└─ example-1.20.1/
   ├─ example-1.20.1.jar
   ├─ キー割り当てと操作方法.md
   ├─ 解析ログ.json
   └─ Codex追記依頼.md
```

## CodexでMarkdownを充実させる流れ

この自動解析は「前処理」です。最終的な読みやすい解説は、生成された `Codex追記依頼.md` を使って Codex に仕上げさせます。

1. jar をダウンロードします。
2. `mod解析自動化.ps1` を実行します。
3. 作成されたフォルダを開きます。
4. `Codex追記依頼.md` の「Codexへの依頼文」を Codex に渡します。
5. Codex に `キー割り当てと操作方法.md` を編集させます。
6. 必要に応じて、実機確認した内容を追記します。

Codexに頼む内容は、毎回だいたい次の方針です。

```text
このフォルダ内の jar、解析ログ.json、キー割り当てと操作方法.md を読み、jarを必要に応じて逆コンパイル・文字列検索・メタデータ確認しながら、キー割り当てと操作方法.md を完成版に近づけてください。
```

## 注意点

- 自動生成されるMarkdownは下書きです。キー名、コマンド候補、関連クラスはかなり拾えますが、実際の発動条件や詳しい遊び方はゲーム内確認が必要です。
- コマンドは class 内の文字列から推定するため、引数構造はゲーム内補完や追加解析で確認してください。
- Java の `javap` が必要です。このPCでは JDK が入っているため、そのまま使える想定です。
- 大きなMODでは解析に少し時間がかかります。
