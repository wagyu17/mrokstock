# Konkrete キー割り当てと操作方法

## MOD概要

Konkrete は、Keksuccino氏のMOD群で使われるライブラリMODです。単体で遊び方を追加するMODではなく、画面・設定・UI系の共通処理を他MODに提供します。

## 解析元

- jar: `konkrete_fabric_1.10.2_MC_26.1.1.jar`
- modId: `konkrete`
- loader: Fabric
- 主な解析対象: `fabric.mod.json`, `de/keksuccino/konkrete/**`

## キー割り当て

通常プレイ用のKonkrete専用キーは検出されませんでした。

解析ログ上では `key.keyboard.` に近い文字列が検出されましたが、これはキー入力処理の内部文字列であり、プレイヤーが設定する専用キーではありません。

## 独自コマンド

内部デバッグ用と思われるコマンドラインデバッグ機能が含まれています。

| コマンド候補 | 内容 |
| --- | --- |
| `/konkretedebug help` | デバッグ機能のヘルプ |
| `/konkretedebug screen` / `/konkretedebug screensize` | 現在の画面や画面サイズの確認 |
| `/konkretedebug widgets` | UIウィジェット情報の確認 |
| `/konkretedebug mousemove` / `/konkretedebug mousedelta` | マウス移動関連のデバッグ |
| `/konkretedebug click` / `/konkretedebug widgetclick` | クリック処理のデバッグ |
| `/konkretedebug key` | キー入力のデバッグ |
| `/konkretedebug pause` | ポーズ画面関連のデバッグ |
| `/konkretedebug leaveworld` / `/konkretedebug leave` | ワールド離脱処理のデバッグ |
| `/konkretedebug title` | タイトル画面関連のデバッグ |

通常プレイや授業で使う必要はほぼありません。

## 操作方法

### プレイヤーとして使う場合

Konkrete は前提MODとして導入するだけで機能します。専用アイテムや専用画面を直接操作する必要はありません。

1. Konkreteを必要とするMODと一緒に導入します。
2. このjarは Fabric 版なので、Fabric環境で使います。
3. 起動時に依存MODが正常に読み込まれれば導入完了です。

### 開発・検証で使う場合

`/konkretedebug` 系は、画面UIや入力処理を確認するためのデバッグ用途です。MOD制作・検証時以外は触らない方が安全です。

## 注意点

- このjarは Fabric 版です。NeoForge環境には基本的に対応しません。
- Konkreteが必要なMODを抜かずにKonkreteだけ削除すると、起動エラーになる場合があります。
- デバッグコマンドはユーザー向け機能ではないため、環境によっては使えない・権限が必要・表示が変わる可能性があります。
