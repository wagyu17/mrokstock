## ワールド設計

### マップ全体構成

ワールドは「建築プログラミングチャレンジランド」として設計する。平坦な草原バイオームに、あらかじめ以下の要素を配置しておく。

**事前配置する要素：**

- **スタートエリア**：石レンガで作った大きなスタート地点マーカー（3×3ブロック）。エージェントがここから開始
- **チャレンジゾーン1〜3**：各チャレンジの実施エリアを色付きウール（赤・青・黄）で枠付けし、視覚的に区別
- **コマンドカード展示エリア**：コマンドカードのイラストが置かれた看板が立つエリア（「前に進む」「右を向く」「ブロック置く」など）
- **完成見本エリア**：各チャレンジの模範解答例を表示（エージェントが作った「正解の形」）
- **失敗体験コーナー**：「エージェントが間違えたらどうなる？」を体験できるエリア（転落・無限ループなど、安全な環境で見学）
- **リセットボタンゾーン**：各チャレンジエリアを一瞬でリセットするボタン（赤石回路）

**マップサイズ：** 約100×100ブロックの範囲に収める。各チャレンジエリア間に十分な余白を持たせ、混乱を防ぐ。

**ワールドボーダー：** 建設予定地の外には見えない壁（バリアブロック）を設定。

### ギミック詳細

#### ギミック1：「3歩進んで、右を向いて、ブロックを置く」→ 順次処理の基礎

**仕掛け：**
- チャレンジゾーン1（赤ウール枠）に、スタートマーカーからゴールマーカーまで約10ブロックの直線距離を用意
- エージェントはスタートマーカー上に配置（North向き）
- ゴール地点の床に、異なる色（金ブロック）でマーク
- PDF教材に3つの命令を順番に実行する図を示す：「前に進む」→「前に進む」→「前に進む」→「右を向く」→「ブロック置く」

**学習ポイント：**
- 命令には「順番」がある。順番が違うと結果が変わる
- 「右を向く」の前に「ブロック置く」をするとどうなる？という実験をする
- エージェントは「指示に従う機械」であり、自分で判断しない

**褒めポイント（先生用）：**
- 「その順番だと、ブロックができたね！」
- 「命令の順番を変えたら、どこにブロックが置かれた？」
- 失敗しても「失敗も大事な勉強だ！何が起きた？」と前向きに

#### ギミック2：「正方形を描く」→ ループ（繰り返し）の概念

**仕掛け：**
- チャレンジゾーン2（青ウール枠）：空の平坦エリア（10×10ブロック）
- エージェントはスタート地点の角に配置
- PDF教材に「16個のブロックを置いて、正方形を作ろう」という課題を提示
- 解法例1：「命令を16個書く」（非効率的だが可能）
- 解法例2：「『前に進む→右を向く→ブロック置く』を4回繰り返す」（ループ）
- 模範解答：「repeat 4 { forward 1, turn right, place block }」

**学習ポイント：**
- 同じ命令を何度も書くのは大変 → ループで書き直すと簡潔
- 「4回繰り返す」という概念の理解
- 正方形が完成する喜び（視覚的な成功体験）

**褒めポイント（先生用）：**
- 「正方形ができた！命令が少ないのに、ちゃんと形になったね！」
- 「何回繰り返したら正方形ができる？」と児童に気づかせる
- 「ループって、魔法みたい。少ない命令で大きいことができる！」

#### ギミック3：「壁を作る、でも端ですぐに曲がる」→ 条件分岐（if/else）の入口

**仕掛け：**
- チャレンジゾーン3（黄ウール枠）：15×5の長方形エリア
- エージェントはスタート地点に配置
- 課題：「端まで行ったら、曲がって、次の列を作ってね」→ 矩形の壁（レンガブロック）
- MakeCodeで「もし〜なら」ブロックを使って実装
  - 例：「もしエージェントが壁に当たったら、右を向く」
  - または「もしエージェントがX座標15に達したら、右を向く」

**学習ポイント：**
- プログラムは「条件」に基づいて判断する
- 「常に同じ命令をする」のではなく、「状況に応じて異なる命令をする」
- 複雑な建築を簡潔に記述できる

**褒めポイント（先生用）：**
- 「難しい形だけど、プログラムを書いたら完成した！」
- 「条件分岐って、プログラムが『考えている』みたいだね」
- 失敗時も「どこで曲がるように書いた？」と原因を一緒に探す

### 使用ブロック一覧

| ブロック名 | 用途 | 事前配置場所 |
|-----------|------|------------|
| 石レンガ | スタートマーカー、チャレンジの枠 | ワールド各所に事前配置 |
| 色付きウール（赤・青・黄） | チャレンジゾーンの標識 | 地面に事前配置 |
| 金ブロック | ゴール地点の標識 | 各チャレンジゾーン |
| オークのブロック | エージェントが配置するメイン素材 | 無限（コマンドで生成） |
| レンガブロック | チャレンジ3の壁材 | チェストに事前配置 |
| 看板 | コマンドの説明、チャレンジの指示 | 各エリアに事前配置 |
| バリアブロック | ワールド境界 | 外周に事前配置 |

## 授業設計

### 学習目標

| レベル | 目標 |
|--------|------|
| 知識 | プログラムの3要素（順次・繰り返し・条件分岐）を理解する |
| 技能 | MakeCodeで簡単な命令を組み合わせプログラムを作成できる |
| 思考 | 複雑な問題を「繰り返す」ことで簡潔に解く発想ができる |
| 態度 | 失敗を恐れず試行錯誤する姿勢、エージェント失敗への肯定的態度 |

### 授業の流れ（90分の場合）

#### 導入（10分）

1. **興味喚起（3分）**：「今日は、ロボットに建築させます。でも、ロボットは何も考えていない。あなたたちが『何をしてほしい』かを命令しないといけません」と説明
2. **デモンストレーション（4分）**：MakeCodeの簡単なプログラムでエージェントを動かし、「ほら、命令通りに動いた！」という驚きと楽しさを引き出す
3. **ワールド説明（3分）**：3つのチャレンジゾーンを見せ、「これらを作ってみようね」と目標を示す

#### 展開1：順次処理（20分）

1. **課題提示**：チャレンジゾーン1を見学。「このエージェントに、スタートからゴールまで行かせて、ブロックを置かせてね」
2. **コマンド説明**：PDF教材のコマンドカードを見ながら、「前に進む」「右を向く」「ブロック置く」の3つを説明
3. **プログラム作成**：先生がMakeCodeを映す。児童が「次はどうするの？」と指示する。先生が命令ブロックを組み立てる（初回は全体で）
4. **実行**：プログラムを実行。成功したら大きく褒める
5. **失敗体験**：「『ブロック置く』を最初にしたらどうなる？」と順番を変えて実行。「命令の順番が大事だ！」と気づかせる

**声かけ例：**
- 「次の命令は何だと思う？」
- 「ブロックが置かれた！順番が正しかったんだね！」
- 「失敗しちゃった。でもこれで順番の大事さがわかったね」

#### 展開2：繰り返し（25分）

1. **課題提示**：チャレンジゾーン2を見学。「4×4の正方形を作ってね。ブロックは16個必要。命令をいっぱい書く？それとも…？」
2. **プログラム作成法1**：「もし16個の命令を書いたら？」→ 実際に書く。児童は見守り、「すごく長い…」と気づく
3. **プログラム作成法2**：「同じ命令を何度も書くのは大変だから『繰り返す』ブロックを使おう」→ MakeCodeの「repeat」ブロックを紹介
4. **ここが重要**：「『前に進む→右を向く→ブロック置く』を4回繰り返すと、正方形になる」と児童と一緒に発見する
5. **実行**：繰り返しプログラムで実行。正方形が完成したら、「ほら！少ない命令で大きいことができた！」と強調

**声かけ例：**
- 「16個の命令は大変だね。これを短くできる？」
- 「『繰り返す』を使うと、何回同じことをさせられる？」
- 「正方形ができた！ほら、繰り返しって便利だ！」

#### 展開3：条件分岐（25分）※段階的難易度対応

**初級（時間に余裕がない場合は省略）：**
- チャレンジゾーン3の簡易版：「壁に当たったら曲がる」という単純な条件

**標準版：**
1. **課題提示**：「長い矩形の壁を作ってね。でも、つかい方はあなたの工夫次第」
2. **プログラム作成**：MakeCodeの「if」ブロックを紹介。「もしエージェントが壁に当たったら、右を向く」
3. **ここが重要**：児童と「どういう条件を書く？」と一緒に考える。「X座標が15になったら」など、複数の選択肢を試す
4. **実行と改善**：失敗したら「どの条件が間違ってた？」と一緒に原因を探す

**褒めポイント（先生用）：**
- 「複雑な形だけど、プログラムで完成させた！」
- 「条件を書き直したら、上手くいった！」

#### まとめ・振り返り（10分）

1. **成功体験の共有**：「プログラムを書いて、ブロックが置かれた。あなたたちがプログラマーだ！」
2. **3つの概念の再確認**：「順番が大事」「繰り返すと便利」「条件を考えるとプログラムが賢くなる」
3. **振り返りシートに記入**：PDF教材の最後のページに「分かったこと」「むずかしかったこと」「つぎやりたいこと」を記入
4. **発表（任意）**：「自分が書いたプログラムを見せたい人？」と声かけ

### 先生用ガイド

#### つまずきやすいポイントと対応

| つまずき | 原因 | 対応方法 |
|---------|------|---------|
| MakeCodeの操作がわからない | ビジュアルプログラミング自体が初めて | 先生が一度大きく映して、「この『青い箱』を『赤い箱』に重ねる」と具体的に示す |
| エージェントが期待と違う動きをする | プログラムのどこが間違っているか判断できない | 命令を1つずつ実行して、「ここで何が起きてる？」と一緒に確認 |
| 「繰り返す」の概念がわからない | 抽象的な説明では理解できない | 「『歩く→曲がる』を4回やったら、正方形になるよ」と、具体的な例で示す |
| 条件分岐が難しい | まだ論理的思考が発達段階 | 無理に進めない。チャレンジ2までで今回は終了し、条件分岐は次回以降に |
| エージェント失敗にネガティブ反応 | 「失敗＝悪いこと」という認識 | 「失敗は大事な発見だ。ここで何が起きた？」と肯定的に捉え直す |
| 早く終わった子が暇になる | 発展課題がない | 次のチャレンジに進ませる。もしくは「オリジナルの形を作ってみて」と自由課題を出す |

#### 評価基準

| 評価観点 | A（十分達成） | B（概ね達成） | C（支援が必要） |
|---------|-------------|-------------|---------------|
| 順次処理の理解 | 命令の順番の重要性に気づき、自分で修正できた | ヒントで順番の大事さに気づいた | 命令の順番の概念がまだ曖昧 |
| ループの活用 | 「繰り返す」を自分から提案し、使いこなせた | ヒントがあれば繰り返しブロックを使えた | 繰り返しの概念が難しく、個別プログラムで対応 |
| MakeCode操作スキル | 自分でブロックをドラッグ&ドロップし、組み立てられた | 先生の指導で操作できた | マウス操作など基本的な支援が必要 |
| 失敗への向き合い | エージェント失敗をポジティブに捉え、試行錯誤した | 失敗に一時的に落ち込んだが、励ましで再挑戦 | 失敗に強く落ち込み、再挑戦が難しかった |

**重要：** C評価であっても「プログラムを作ろうとしたこと」「1つのブロックでも組み立てたこと」を必ず褒める。この教材の最大目的は「自分で指示を出す体験」と「失敗しても大丈夫」というマインドセット構築である。

## PDF教材構成案

### ページ構成（全10ページ）

| ページ | 内容 | 備考 |
|--------|------|------|
| 1 | 表紙「エージェントと建築」 | イラスト付き（ロボットがブロック置く様子） |
| 2 | 「プログラムって何？」の説明 | 「命令を順番に書いたもの」と簡潔に説明 |
| 3 | コマンドカード図解 | 「前に進む」「右を向く」「ブロック置く」の3つを図示 |
| 4 | チャレンジ1：順次処理 | スタート→ゴールの図、命令の順番を学ぶ |
| 5 | チャレンジ1の解説 | 「順番が違うと、結果が変わる」の例示 |
| 6 | チャレンジ2：ループ（繰り返し） | 正方形の図、「4回繰り返す」の説明 |
| 7 | チャレンジ2の解説 | 「繰り返しは便利」「命令が短くなる」 |
| 8 | チャレンジ3：条件分岐（if） | 矩形の図、「もしそこに着いたら」の説明 |
| 9 | プログラミング用語集 | 「順序」「繰り返し」「条件」などの用語をわかりやすく |
| 10 | ふりかえりシート | わかったこと・むずかしかったこと・つぎやりたいこと |

### デザイン指針

- フォント：ゴシック体、16pt以上（ただし、プログラミング用語は等幅フォント）
- ふりがな：すべての漢字にふりがな付き
- 色使い：プログラミング教科カラー（紫系）をアクセントに、コマンドカードは色分け（青＝移動、赤＝設置など）
- イラスト：エージェントの動きを矢印で視覚化。成功時は星やハート、失敗時は「？」で表現
- 記入欄：大きめに取り、鉛筆で書きやすいサイズ
- 余白：たっぷり取り、圧迫感を出さない
- QRコード：MakeCodeのテンプレートへのリンク（オプション）

### コマンドカードリファレンス（PDF別紙）

**推奨：A6サイズで印刷し、児童が実際に組み立てながら学べるようにする**

| カード | 説明 | MakeCode命令 |
|--------|------|------------|
| 前に進む | エージェントが1ブロック前に移動 | agent.move(Forward, 1) |
| 後ろに進む | エージェントが1ブロック後ろに移動 | agent.move(Back, 1) |
| 右を向く | エージェントが右に90°回転 | agent.turn(TurnRight) |
| 左を向く | エージェントが左に90°回転 | agent.turn(TurnLeft) |
| ブロック置く | エージェントが足元にブロックを配置 | agent.place(Down) |
| ブロック壊す | エージェントが足元のブロックを破壊 | agent.destroy(Down) |

## オプション：発展学習

### 応用課題（時間に余裕がある場合）

1. **「複数色で虹を描く」**：異なる色のブロックを順番に置いて、虹色の壁を作る
2. **「複雑な図形に挑戦」**：五角形、六角形など、繰り返しの回数を変えて作成
3. **「ゲーム化」**：「エージェントを目的地に導く」パズルゲームとして遊ぶ

### 発展的な学習内容（次のステップ）

- **変数の導入**：「ブロック数を数える」など、変数で数を管理する
- **関数の導入**：「正方形を描く」を1つの関数にして再利用
- **複雑な条件**：「AND」「OR」を使った複数条件の組み合わせ
- **リスト・配列**：複数の値を管理して、複雑なパターンを描く
