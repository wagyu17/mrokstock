## 技術制約と判明した仕様

### 制約

| 制約 | 理由 |
|------|------|
| MakeCodeはワールドギミックに使用不可 | インポート時に自動実行されないため |
| コマンドブロック・ストラクチャーブロック・NPCのみで完結 | スタッフがインポートして即プレイできる必要がある |
| MakeCodeはプログラミング科目の授業内容としては使用可 | 児童が授業中に操作するため問題なし |

### Education Edition 1.21.132 で判明した仕様

| 仕様 | 詳細 |
|------|------|
| `.mcfunction` は利用可能 | ビヘイビアパックの `functions/` フォルダに配置 |
| コマンドブロックのブロックステート指定不可 | `/setblock` で `facing_direction` / `conditional_bit` を設定できない |
| ブロックID フラット化（1.19〜1.21） | `fence`→`oak_fence` / `stonebrick`→`stone_bricks` / `concrete ["color":"lime"]`→`lime_concrete` など |
| ブロックステート記法 | 1.20.10以降 `":"` → `"="` |

#### 💡 コマンドブロック設定不可（`setblock`）に対する代替案
上記「コマンドブロックのブロックステート指定不可」に対するワークアラウンドです。

1. **`/structure` コマンドを使用（推奨）**
   - あらかじめ向きや条件を設定したコマンドブロック（内部のコマンドやNBTデータ含む）をストラクチャーとして保存しておき、`/structure load` で呼び出す方法。
2. **`/clone` コマンドを使用**
   - ワールド内の隔離エリアに、あらゆる向き・条件パターンのコマンドブロックをパレットのように用意しておき、`/clone` で必要な場所に持ってくる方法。
3. **連鎖機構自体を廃止し `.mcfunction` に処理をまとめる**
   - 物理的なコマンドブロックの連鎖をワールド内に配置するのをやめ、処理をすべて関数内に記述。条件分岐は `execute if ...` やスコアボードで代用します。

### Behavior Pack（ビヘイビアパック）の仕様・ルール (1.21準拠)

現在のまとめの記載（`functions/` フォルダに配置、`manifest.json` が必要 等）は正しいですが、実際の開発においては**統合版特有の以下の厳密なルールの順守・留意が必要**です。まとめとの「差分・補足」として以下を追記しました。

| ルール / 制約 | プロジェクトへの適用・留意点 |
|--------------|--------------------------|
| **UUIDの重複不可** | `manifest.json` の `header` と `modules` には、それぞれ**全く異なる2つのUUID**（v4等）を発行して記述する必要があります。 |
| **ファイルの文字コード** | `manifest.json` および `.mcfunction` ファイルはすべて **UTF-8（BOMなし）** で保存する必要があります。BOMありだと読み込みエラーになります。 |
| **コマンド先頭のスラッシュ (`/`)** | `.mcfunction` 内の各コマンドの先頭には `/` を**付けないのが標準的**です。（シンタックスエラーを防止するため省略推奨） |
| **1チックの実行上限** | 1回の関数呼び出しで実行できるコマンド群は、ネストを含めて最大 **10,000行** までという上限があります。 |
| **Java版との構文差** | `execute` の構文は 1.19.50 以降 Java 版とほぼ統合されましたが、NBTデータの直接検知や操作など一部機能には未だ制限・仕様の違いがあります。 |

## Claude Code でできること

### ✅ 完全自動化できる

| 作業 | 方法 |
|------|------|
| 部屋・建築物の構造生成 | `.mcfunction` ファイルに `fill` / `setblock` / `summon` を記述 |
| ビヘイビアパックの構成ファイル生成 | `manifest.json` をコード生成 |
| ブロックID・コマンド構文のバージョン対応 | 仕様調査→修正をすべて対応可 |
| 設計書・仕様書・マニュアルのMarkdown作成 | テキストファイルとして出力 |
| 生徒用PDF・スタッフ用PDFの原稿作成 | Markdownで下書き生成（PDF化は別途） |
| コマンド設定書（座標計算・コマンド一覧） | 座標を教えてもらえれば即座に出力 |
| Stage 2以降の設計・mcfunction生成 | テンプレートから量産可能 |
| NPCセリフの文章作成 | 学年・特性に合わせた文体で生成 |

### ⚠️ 一部自動化できる（追加情報が必要）

| 作業 | 必要な情報 |
|------|-----------|
| コマンドブロックへの入力コマンド生成 | 実際の設置座標（F3で確認後に共有） |
| `/structure save` コマンド生成 | 最終的な部屋の原点座標 |
| テレポート先座標の設定 | 各部屋の設置座標 |

### ❌ Claude Code ではできない（Minecraft内の手動作業）

| 作業 | 理由 |
|------|------|
| コマンドブロックの設置・向き設定 | ブロックステートを `/setblock` で指定できない仕様 |
| コマンドブロックへのコマンド入力 | GUI操作が必要（テキスト設定コマンドが存在しない） |
| コマンドブロックのConditionaltoggle | GUI操作が必要 |
| NPCの設置・ダイアログ設定 | Minecraft内の操作が必要 |
| 看板へのテキスト入力 | GUI操作が必要 |
| ストラクチャーブロックの保存操作 | Minecraft内の操作が必要 |
| 動作テスト・デバッグ | 実機確認が必要 |
| ワールドのビジュアルデザイン調整 | 実際に見ながら決める作業 |
