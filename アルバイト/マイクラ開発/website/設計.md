# ONELIFE ドリル＆プログラミング — Website 設計概要

## 1. サービス概要

**対象ユーザー**: 小学生（タブレット・PC操作を前提）
**目的**: 算数・プログラミング・タイピングの基礎学力を、Minecraft風のゲーム体験を通じて楽しく定着させる
**技術スタック**: バニラHTML/CSS/JS（フレームワーク不使用）・LocalStorage・Canvas API
**SEO方針**: `noindex, nofollow`（外部公開を想定しない閉じたサービス）

---

## 2. ナビゲーション構造

```
ホーム
├── 算数
│   ├── 足し算（3段階難易度）
│   ├── 引き算（4段階難易度）
│   ├── 掛け算（1〜9の段 + ランダム20問タイムアタック）
│   └── 割り算（4段階難易度、余りあり/なし）
├── プログラミング
│   └── ミッション1〜5
└── タイピング
    ├── ワードディフェンス
    └── 君が代タイムアタック
```

各フローは「メニュー → ゲーム → リザルト」の3画面で完結する。

## 3. モジュール詳細

### 3-1. 算数ドリル

**問題生成**: `buildProblems()` が難易度パラメータに応じて問題を自動生成。

| 演算 | 特徴 |
|------|------|
| 足し算 | 難易度別に数値範囲を制限。ブロックのグループ表示で視覚化。 |
| 引き算 | 差が負にならないよう生成。色分けブロック表示。 |
| 掛け算 | 10×10マトリックスでブロック数を視覚化。タイムアタック対応。 |
| 割り算 | 余りあり/なし両対応。グループボックス＋余り強調表示。 |

**フィードバック**:
- 正解: 緑フラッシュ＋「⛏ 一発せいかい！」トースト＋XPオーブ演出
- 不正解: 赤フラッシュ＋シェイク演出（バツ印・否定音なし）
- 進捗: プログレスドット（画面下部）

### 3-2. プログラミングミッション

Canvas上でロボット(🤖)をチェスト(🎁)まで誘導するビジュアルプログラミング。

| 命令ブロック | 効果 |
|------------|------|
| 1歩進む | 向いている方向に1マス移動 |
| 右を向く | 方向を時計回りに90度回転 |
| 左を向く | 方向を反時計回りに90度回転 |
| 繰り返す（2〜4回） | 直後の命令を指定回数ループ |

**ミッション構成**:
1. 直進操作
2. 右折応用
3. 障害物回避
4. ループ構文活用
5. 複雑な経路（ジグザグ）

実行時は30FPSアニメーションで1マスずつ動作を確認でき、どこで誤ったか直感的にわかる。

### 3-3. タイピング

#### ワードディフェンス
- 上から落下するブロック型単語をタイピングして「撃退」
- 難易度3ティア（短い単語 → 長いカタカナ語へ）
- コンボシステム（連続正解でスコア倍増）
- ライフ3ハート制。時間経過で落下速度が上昇。

#### 君が代タイムアタック
- 「君が代」全文をいかに速く正確に打つかを競う
- 石版(🗿)に文字が彫り込まれるアニメーション演出
- ひらがな＋ローマ字を大きく併記（読みがなで国語教育も兼ねる）
- クリアタイムを記録・表示

## 4. UI/UXデザイン方針

### ビジュアルテーマ
- **Minecraft風ピクセルアート**をCSSのみで再現（画像ファイルゼロ）
- カラーパレット（CSS変数）:

| 変数名 | 色 | 用途 |
|--------|-----|------|
| `--grass-top` | #7cc34a | 草ブロック・肯定色 |
| `--stone` | #9a9a9a | 石ブロック・UIベース |
| `--diamond` | #5ce9e6 | ダイヤモンド・最高ランク |
| `--gold` | #fbd24a | 金・強調 |
| `--redstone` | #d93b2a | レッドストーン・警告 |

- フォント: `DotGothic16`（メイン）、`Press Start 2P`（スコア数値）

### アクセシビリティ
- ひらがな中心・ルビ対応で漢字への不安を軽減
- タブレット操作前提の大きなボタン・広い間隔
- レスポンシブ対応: `@media (max-width: 640px)`

### 否定しないフィードバック原則
- 赤いバツ印・否定音は一切使わない
- 「おしい！」「もういっかい！」など再挑戦を促す言葉に統一

## 5. ランク・報酬システム

### ランク（総ポイント累計）

| ランク | 必要ポイント |
|--------|------------|
| 木 | 0P |
| 石 | 500P |
| 鉄 | 2,000P |
| 金 | 5,000P |
| ダイヤ | 10,000P |
| エメラルド | 20,000P |
| ネザライト | 40,000P |

### ポイント計算（算数）

```
スコア = 正解数 × 100 + 完璧回答数 × 50 + 100 + タイムボーナス
タイムボーナス = max(0, 300秒 − 実時間) × 10  ※タイムアタック時のみ
```

### メダル条件

| メダル | 条件 |
|--------|------|
| 金 | 全問1発正解（パーフェクト） |
| 銀 | 完璧回答率70%以上 |
| 銅 | 全問正解（クリア） |

## 7. ファイル構成

```
website/
├── index.html     — 全画面のHTMLマークアップ（SPA構造）
├── style.css      — ピクセルアート・テーマ・アニメーション（40+キーフレーム）
├── script.js      — 問題生成・ゲームロジック・Canvas描画・LocalStorage管理
├── robots.txt     — Disallow: /（全クロール拒否）
└── 設計.md        — 本ドキュメント
```

`index.html` は複数の `view-*` セクションをCSSの表示切り替えで制御するSPA設計。ページ遷移なしで全画面を提供する。

## 8. 元設計方針（原文）

以下は開発当初に策定した設計方針の原文を参考として残す。

---

### サイトの全体構造と導線設計

階層を浅くし、直感的に目的のドリルへたどり着ける構造にします。

- トップ画面: 「算数」または「プログラミング」の大きな選択ボタン
- 算数エリア: 「足し算」「引き算」「掛け算」「割り算」の選択
- 掛け算エリア:
  - 基礎ステージ: 「1の段」〜「9の段」の9ボタン
  - 応用ステージ: ランダム20問早解き（タイムアタック）

### UI/UXとアクセシビリティ方針

- 全ボタン・枠線を「ピクセルアート」で統一
- テキストを大きくし、漢字にはルびを振るかひらがな中心
- タブレットでのタップを前提に、ボタン間隔を広く
- 否定しないフィードバック: 「おしい！」「もういっかい！」
- 問題解答→次問題表示のラグをなくし、テンポの良さを最優先

### モチベーションと継続学習の設計

- スモールステップの可視化: 段ごとにメダル・星マークを付与
- ランダム20問早解きのゲーム化: 自己ベスト更新時に派手な演出
- 実体験への橋渡し: 次のステップへの自信につながる仕組み

### プログラミングモジュール（アルゴリズム学習）

- ビジュアルプログラミング: Scratch風の命令ブロック
- ミッション: 「障害物を避けてチェストまでロボットを動かせ」
- デバッグ機能: 1マスずつ実行し、誤りを直感的に把握

### タイピングゲーム設計方針

**ワードディフェンスモード**
- 落下ブロック（単語）をタイピングして撃退するサバイバル感覚
- コンボボーナス、難易度自動上昇、ライフ制

**君が代タイムアタック（石版彫刻モード）**
- 石版に文字が彫り込まれる演出
- ひらがな・ローマ字の両方を大きく併記
- 国語的・文化的な素地学習として機能
