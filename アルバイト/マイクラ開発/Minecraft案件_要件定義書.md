# 株式会社ワンライフ 放課後等デイサービス向け Minecraft学習コンテンツ 要件定義書（第6版：認知機能・PDF整合版）

---

## 1. プロジェクトの目的と理念

### 根本的な問い
「勉強は何のためにするのか」を子供自身が体感で理解することが本プロジェクトの出発点である。

### 中心思想
勉強を「させられるもの」ではなく、**「目標（ミッション）を達成するための道具」** として子供自身が気づく体験を設計する。マインクラフト内で「自分の街（前橋）をよくしたい」「問題を解きたい」という意欲が先に生まれ、その手段として算数・理科・国語などの知識が必要になる構造を作る。

> 例：「前橋駅前に公園を設計したい」→ 面積の計算が必要になる → 「算数って役に立つんだ」

### 対象者
- **メイン対象**: 発達特性のある小学生（1〜6年生）
- **拡張対象**: 中学生・高校生（難易度「上級」ミッションで対応）
- **施設所在地**: 群馬県前橋市

### 学習サイクル（8ステップ）
全ミッションに共通する探求学習の構造（探求学習の手順との対応）：

```
① 操作手順の説明
② 操作手順の実施
③ 簡単な課題の提供　（課題設定）
④ やり方の確認　　　（情報収集）
⑤ 実施　　　　　　　（整理・分析）
⑥ 完成　　　　　　　（まとめ・表現）
⑦ 振り返り
⑧ 自由課題
```

> ①②はマイクラ操作習得済みの場合は省略可。③～⑧が探求学習の中核サイクルとなる。

### 「考え・試し・直し・伝える」と認知機能の発達

本プログラムの設計は、発達特性のある子供の**高次認知機能の発達**を直接支援することを根拠としている。各ステップは以下の認知機能に対応する。

| ステップ | 対応する認知機能 | 発達する力 |
|---|---|---|
| **考え**（③④） | 組織化・計画（高次認知機能） | 課題を整理し、順序立てて取り組む力 |
| **試し**（⑤） | 実行機能 | 考えを行動に移す力・主体的に取り組む姿勢 |
| **結果をみて**（⑥） | 判断・洞察・問題解決 | 結果から問いを自分で生み出す力 |
| **直し**（⑦） | 認知の柔軟性 | 自分の考えを自分で切り替える力・学習意欲の創出 |
| **伝える**（⑦⑧） | 自己表現・コミュニケーション | 言葉で表出する力・SOS発信・ストレス対処技能 |

加えて、チームでのミッション実施を通じて以下の力も育む：
- **他者と協力する力**：役割分担・意見調整・ルール遵守による社会性の獲得
- **自己調整力**：活動時間の制限・ルールを守る・終了時の気持ちの切り替えなど、楽しさの中で育つ自己コントロール

## 2. システム要件と成果物

### プラットフォーム
- **Minecraft Education**（教育版マインクラフト・ライセンス契約済）
- 全コンテンツはビジュアルプログラミング（MakeCode Agent）との連携を前提とする

### 必須成果物（2点セット）

| 成果物 | 内容 | 形式 |
|---|---|---|
| **Minecraftカスタムワールド** | 前橋市実地形ベースのワールド1本。5ミッション×3難易度（計15エリア）をテレポートで接続 | `.mcworld` × 1ファイル |
| **連動型PDF教材** | 各ミッションに対応した指導者向けマニュアル・児童向けワークシート | 各ミッションにつき1セット（計15セット） |

### 地形ベース技術
- **Arnis**（OpenStreetMapデータ活用ツール）を使用して群馬県前橋市の実地形をMinecraftワールドに変換
- **再現済み**: 前橋駅周辺エリア
- **活用方針：没入感のある舞台背景として使用する**
  - 子供たちが「自分の街っぽい」と感じる空間として探検・ミッションの舞台に活用する
  - 地形・建物の精度は地理教材として使用できるレベルではないため、**地図学習・町探検の代替教材としては使用しない**
  - 地理・社会科の学習内容（地図記号・ハザードマップ・建物配置等）はPDF教材や手動で作成したワールド内構造物で補う
  - 前橋のランドマーク（駅・市役所等）はArnis地形の上に手動で建設し、認識しやすくする

## 3. ワールド構造

### 全体構造

```
[前橋市実地形ベースワールド（Arnis生成）]
                  │
      ┌───────────┴───────────┐
      │     ハブ拠点            │
      │  （前橋駅周辺エリア）    │
      │  ・チュートリアル神殿    │
      │  ・マイルーム           │
      └───────────┬───────────┘
                  │
    ┌─────┬────┬─────┬──────┐
    ▼     ▼    ▼     ▼      ▼
まちを  自然を  ものを  ことばで  せかいを
つくろう 探ろう  つくろう 伝えよう  のぞこう
```

### ミッション構成（5カテゴリ × 3難易度）

| ゲート | やさしい（小1〜3相当） | ふつう（小4〜6相当） | むずかしい（中高相当） |
|---|---|---|---|
| **まちをつくろう** | 前橋の公園に花壇を設計しよう | 前橋駅前広場をリデザインしよう | 商店街の空き地を再開発しよう |
| **自然を探ろう** | 赤城山で植物と動物を観察しよう | 利根川の洪水から街を守ろう | 前橋の気候と農業の関係を調べよう |
| **ものをつくろう** | 木の橋で川を渡れるようにしよう | 壊れない家を設計しよう | 太陽光で電力を賄う建物を作ろう |
| **ことばで伝えよう** | 前橋の案内看板を作ろう | 前橋の観光マップを完成させよう | 市長への街の提案書を書こう |
| **せかいをのぞこう** | 前橋と別の街の違いを探そう | ロンドンのまちづくりを探検しよう | NYCと前橋を比較して提案しよう |

> **「せかいをのぞこう」（上級）**: 他都市（ロンドン・NYC等）の地形データをArnisで追加生成し、テレポートで探検できる構成とする（開発フェーズ3以降に着手）

### 教科との対応関係（参考）
各ミッションに自然な形で教科的知識が組み込まれる。教科は「目的」ではなく「手段」として機能する。  
地理・社会科の学習内容はArnis地形に依存せず、**PDF教材・手動構造物で補完する**。

| ミッションの例 | 自然に使われる知識 | 地形依存度 |
|---|---|---|
| 公園に花壇を設計しよう | 面積・図形（算数）、植物の特性（理科） | 低（舞台として使用） |
| 利根川の洪水から街を守ろう | 水の流れ（理科）、ハザードマップ（社会）、距離計算（算数） | 低（川・地形はArnis活用、数値・地図はPDF補完） |
| 前橋の観光マップを完成させよう | 地図記号（社会）、文章構成（国語） | 低（ランドマークは手動建設） |
| ロンドンを探検しよう | 国際地理（社会）、英語表記（英語） | 低（雰囲気体験が目的） |

## 4. 機能要件とゲーム設計

### 8ステップの実装方針

- **ステップ①②（操作習得）**: ハブ拠点内のチュートリアルコースで全ミッション共通の操作を習得。操作習得済みの場合は省略可
- **ステップ③（課題設定）**: ミッション開始時にNPCまたは看板で「何を作るか・解くか」を提示。この段階で達成感を得やすい難易度に設定し「できた」体験につなげる
- **ステップ④（情報収集）**: 課題達成に必要な情報（学習ポイント）をPDF教材または看板・NPCで提供する。**教科の知識（算数・国語等）は、この課題設定・指示書の中に「学習ポイント」として明示的に組み込む**
- **ステップ⑤（整理・分析）**: 得た情報をもとに実際にマイクラで実施。「ブロックをここに置くべきか」と思考・判断しながら進め、想像と結果のズレを体感させる
- **ステップ⑥（まとめ・表現）**: 作品の完成から達成感・充足感を得ることが目標。スタッフは完成物を肯定的に評価する（正のフィードバック）
- **ステップ⑦（振り返り）**: 本人の満足度を確認し、うまくいかなかった箇所は「改善点」として前向きに伝える。**「伝える」ことで言葉による記憶の定着・自己表現・SOS発信能力の育成につなげる**
- **ステップ⑧（自由課題）**: ③〜⑦の過程を子供自身が考えて実施。**初めに紙ベースで③〜⑦を書き出してから実施**することで、目標の明確化・書字動作・思考の整理・プログラミング手順の学習を促進する

### 課題設定（指示書）への教科知識の組み込み方

各ミッションのステップ④において、教科的知識は「学習ポイント」として課題設定（指示書）に**事前に明示的に組み込む**。子供が自然に使うだけでなく、使うべき知識を意識できるようにする。

**実装例（「前橋の公園に花壇を設計しよう」の場合）：**

```
【ミッション指示書】
花壇を作ろう。条件：正方形、1辺4ブロック、花は赤と黄を交互に並べること

【学習ポイント】
・正方形とは？（4つの辺の長さが全部同じ四角形）
・1辺4ブロックの正方形には、何ブロック必要？（外周：4×4－内側）
・「交互に並べる」とはどういう意味？（国語：指示の読み取り）
```

教科知識は「ミッションをクリアするための道具」として自然に機能させる。

### アドベンチャーモード保護
- ワールド全体をアドベンチャーモードで保護し、誤操作による破壊を防ぐ
- 指定エリア内でのみ建築可能（`CanPlaceOn` 属性による制御）
- 「失敗しても大丈夫」な設計：正解判定ボタンを押すまでやり直し可能

### テレポート設計
- ハブ拠点に5つのミッションゲートを設置
- 各ゲートに「やさしい・ふつう・むずかしい」の3つの入口
- テレポート先から戻るボタンで必ずハブに戻れる（迷子防止）
- スタッフが操作不要：子供が自分でゲートに入る形式

## 5. 非機能要件（UXと運用）

### スタッフ要件
- **前提**: 指導スタッフはゲーム・PCスキルゼロ
- **運用設計**: スタッフが操作するのは「ワールドを開く」「ワールドを閉じる」の2操作のみ
- **PDF教材**: スタッフが読むだけで授業進行できるスクリプト形式で設計

### 失敗を恐れない環境
- ペナルティなし：何度でもやり直せる
- 否定しない：間違えた場合はNPCがヒントを提示するのみ
- 達成感の積み重ね：小さなクリアを連発できる設計

### データ管理
- `.mcworld` 1ファイルで全ミッションを提供
- OneDrive等クラウドバックアップとの連携でセーブデータの持ち運びを可能にする（検証予定）
