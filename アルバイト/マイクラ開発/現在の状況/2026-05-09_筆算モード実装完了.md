| `website/index.html` | 算数メニューに 5 つ目のボタン「🧾 ひっさん（NEW!）」、3 つの新規 view（`view-hissan-menu` / `view-hissan-stages` / `view-hissan-game`）を追加 |

| `website/script.js` | `HISSAN_OP_META` / `HISSAN_STAGES`（足×3・引×4・掛×2・割×3 = 計 12 ステージ）、状態機械、レンダラ、両替カスケードアニメ、結果画面の再利用ロジック |

#### 4 演算の対応状況
- **足し算**：1 桁ずつ「a + b + (くりあがり) = ?」を入力。桁上がりは次列に伝播表示。
- **引き算**：`effectiveTop`（両替後の固定値）と `coinCounts`（支払いで減る枚数）を分離管理。`top < bot` のとき「🪙 りょうがえ する！」ボタン提示 → `triggerBorrow()` が `100→10×10→10×1` の連鎖両替を非同期演出（光らせ＋✦ バースト）。`504-6` のような連鎖くりさがりも視覚化される。
- **掛け算**：1 桁の乗数に対して各位で `t × m + carry` をテンキー入力。
- **割り算**：長除法の手順を 1 ステップずつ。先頭ゼロ商はサイレントスキップ、開始後の途中 0 は入力させる教育的設計。あまりがあれば結果トーストに `あまり N` を表示。

#### 共通仕様
- 入力はテンキー（タッチ対応）。3 回間違えると正解ヒント。
- スコア：`正解×120 + 1発正解×60 + 100`、`localStorage(onelife_drill_v2)` の `store.best/cleared` に保存。
- 既存ホーム画面の進捗バー／サイドバー `さんすう N/M` バッジに自動反映（`totalMathStages` を `STAGES_BY_OP + HISSAN_STAGES` に拡張済）。
- リザルト画面は既存 `view-result` を `state.resultMode = 'hissan'` フラグで再利用。

## 2. 既存機能との互換性
- 通常ドリル（足／引／掛／割）、プログラミング、タイピング、社会、地図描画モジュールに**影響なし**。
- ステージ ID は `h_` プレフィックスで衝突回避。
- `[data-go="hissan"]` / `[data-go="hissan:add"]` 等のルーティングを既存ハンドラに追加。
