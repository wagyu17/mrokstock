## 2. 設計面の進捗

### 2-1. プロジェクト哲学の確立（04-21）

**ゲートウェイ哲学**を全体方針として確立した。

> 本プロジェクトは完結した学習コンテンツではなく「火付け役（ゲートウェイ）」である。15ミッション全踏破が目的ではなく、児童が自律的な遊び（サバイバル／マルチプレイ／YouTube再現）に向かうための**素地と自信**を育てる。

これに伴い、完成計画書・提案書の論調を「学習網羅型」から「火付け役型」に全面改訂。クライアントにも正面からこの方針を伝える方針に転換した。

### 2-2. 教科構成の確定（04-21）

**メイン4教科 + クロスオーバー1ゾーン = 15ミッション**で確定。

| エリア | 内容 | ミッション数 |
|--------|------|------------|
| 算数 | 四則演算・分配・立体 | 3 |
| 理科 | 元素・ポーション調合 | 3 |
| 社会 | 地理・特産品・地図の見方 | 3 |
| プログラミング | MakeCode | 3 |
| 物理ラボ（算数×理科） | トロッコで速度・運動E体感 | 3 |
| **合計** | | **15** |

国語はNPC会話・看板・クエスト文で全ミッション素地統合。英語は卒業エリア／PDF巻末の発展接続（NASA・Cambridge）で担当。ブリーフィングの「5教科網羅」とは意図的に乖離させ、発達特性配慮とマイクラ世界観整合を優先。

### 2-3. 技術構成・運用方針の決定

| 項目 | 方針 |
|------|------|
| ワールド構成 | **1つの.mcworld** にハブ＋全エリア統合（没入感優先） |
| プレイモード | アドベンチャー固定／卒業エリアのみ自由建築 |
| 進捗管理 | 1人1アカウント＋スコアボード自動記録（セーブ式） |
| マルチプレイ | なし（PC毎個別プレイ） |
| 難易度 | 易・中・上の3段階 |
| エディション | Minecraft Education（既決） |
| 契約主体 | 株式会社ワンライフ（法人） |

| 連絡経路 | **Chatworkグループ**で実施 |

## 8. 所感

一方、ライセンス・PC調達等の運用基盤系はクライアントヒアリング待ちで止まっており、5月初旬のChatworkでの一括ヒアリング起票が次の重要アクション。
