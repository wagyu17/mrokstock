# 個人開発：MacとWindowsでフォルダを同期する（GitHubの活用）

GitHubを使って、MacとWindows間でローカルフォルダを同期させる方法は、個人開発において最も便利で強力な使い方の一つです。

基本的なイメージは、**「GitHubというクラウド上の金庫（リモートリポジトリ）を経由して、両方のPCのデータを最新に保つ」**というものです。

---

## 1. 全体的な仕組み（イメージ）

直接MacとWindowsを通信させるのではなく、間にGitHubを挟みます。

*   **作業の終わり（PC A）：** 変更を `git push` して、GitHubの金庫を最新にする。
*   **作業の始め（PC B）：** GitHubから `git pull` して、PC Bの中身を最新にする。

常にこの「ハブ（GitHub）」を経由してデータをやり取りします。

---

## 2. 最初の準備：もう一つのPCに同期する（初回のみ）

すでに片方のPC（例：Windows）からGitHubにコードを `push` してあると仮定します。
もう片方のPC（例：Mac）に同じフォルダを持ってくるには、ダウンロード（clone）を行います。

**新しく作業したいPCでの操作：**
1. ターミナル（またはコマンドプロンプト）を開く。
2. フォルダを置きたい場所（デスクトップなど）に移動する（`cd Desktop`など）。
3. 以下のコマンドを実行して、GitHubからフォルダごとコピーしてきます。
   ```bash
   git clone https://github.com/あなたのユーザー名/リポジトリ名.git
   ```
これで、両方のPCに同じフォルダが用意されました！

---

## 3. 同期のための「絶対のルール」（毎回の作業）

両方のPCにフォルダが用意できたら、以降は以下のルールを**必ず**守るだけで完璧に同期されます。

### 🟢 作業を【始める】とき（超重要！）
PCを開いて作業を始める前に、**必ず**最新のデータをGitHubからダウンロードします。

```bash
# 作業するフォルダに移動してから
git pull
```
*💡 これを忘れて古い状態のまま編集を始めると、後で「どっちが新しいコード！？」とGitがパニックを起こします（コンフリクト）。*

### 🔴 作業を【終わる】とき
その日の作業（またはそのPCでの作業）が終わったら、**必ず**GitHubに変更をアップロードしておきます。

```bash
git add .
git commit -m "Windowsで〇〇の機能を修正"
git push
```
*💡 コミットメッセージに「Windowsから」や「Macで作業」と書いておくと、後で履歴を見たときに分かりやすいです。*

---

## 4. もし「git pull」を忘れて作業してしまったら？（競合の解決）

「Macで作業してpushしたのに、Windowsでpullするのを忘れて作業してしまった！」
この状態でWindowsから `git push` しようとすると、「GitHub上の方が新しいデータがあるからpushできないよ！」とエラー（Rejected）が出ます。

**対処法：**
1. まず、落ち着いて最新のデータを合流させます。
   ```bash
   git pull
   ```
2. Gitが「自動で合流できたよ」と言えばそのまま `git push` して解決です。
3. もし「同じファイルの同じ場所が編集されていて自動で合流できない（コンフリクト）」と怒られたら、ファイルを開くと `<<<<<<<` などの記号が入っているので、正しい方のコードだけを残して記号を消します。
4. 修正したら、再度保存してアップロードします。
   ```bash
   git add .
   git commit -m "競合を修正"
   git push
   ```

---

## 5. MacとWindows間の同期で気を付けること

OSが違うPC間で同期する場合、以下の2点に注意するとトラブルが減ります。

1. **Mac特有のゴミファイル（.DS_Store）を除外する**
   Macはフォルダを開くだけで `.DS_Store` という見えないファイルを作ります。これがWindowsに同期されると邪魔なので、一番上の階層に `.gitignore` というファイルを作り、以下のように書いておきます。
   ```text
   .DS_Store
   ```

2. **改行コードの違い（CRLF と LF）**
   Windows（CRLF）とMac（LF）で改行の内部的なルールが違います。最近のGitは自動でいい感じに変換してくれますが、もし「何も変更してないのに全部のファイルが変更されたことになっている！」となったら、改行コードのせいです。（基本的にはGitの初期設定で自動変換がオンになっているはずなので、あまり気にしなくても大丈夫です）。
