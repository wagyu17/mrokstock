# ローカル家計簿

MoneyForward系の個人用MVPです。自分のPC内で開ける静的Webアプリとして作っています。

## 起動

`index.html` をブラウザで開きます。

```powershell
Start-Process .\index.html
```

ローカルサーバーで開く場合:

```powershell
node .\server.mjs
```

その後、`http://localhost:4177` を開きます。

## できること

- 現金、三井住友銀行、三菱UFJ銀行、PayPay、モバイルSuicaの管理
- 収入、支出、口座間振替の登録
- 現在の総資産、月ごとの収支、カテゴリ別支出、口座別残高、最近の取引、推移グラフの表示
- 取引一覧とカレンダーモードの切り替え
- CSVインポート
- UTF-8 / Shift_JIS CSVの読み込み
- 店名・内容に基づく自動カテゴリ分類
- 取引一覧からの複数カテゴリ編集
- 取引一覧でのカテゴリ追加
- 銀行出金とPayPay同額入金の自動振替化
- PayPayの100円以下かつ10の倍数でない入金をポイント付与として自動分類
- モバイルSuica残高ご利用明細PDFの取込
- JSONバックアップの書き出し・読み込み

## CSV形式

以下のようなヘッダーを自動判定します。

- 日付: `日付`, `利用日`, `取引日`, `入出金日`, `date`
- 金額: `金額`, `利用金額`, `支払金額`, `amount`
- 入金/出金: `入金`, `出金`, `income`, `expense`, `deposit`, `withdrawal`
- 残高: `差引残高`, `取引後残高`, `現在残高`, `残高`, `balance`
- 内容: `店名`, `利用店`, `加盟店`, `内容`, `摘要`, `取引内容`, `description`
- 種別: `種別`, `入出金`, `取引区分`, `type`
- 口座: `口座`, `account`

以下のファイル名は口座を自動選択します。

- `Transactions_*.csv`: PayPay
- `meisai*.csv`: 三井住友銀行
- `数字_数字.csv`: 三菱UFJ銀行

`sample-import.csv` を取り込みテストに使えます。

`差引残高` などの残高列があるCSVでは、取込後に対象口座の残高を明細の最終残高と照合します。初期残高は自動変更せず、差がある場合は取込結果に警告として表示します。

## データ保存

データはブラウザの `localStorage` に保存されます。同じPCでも別ブラウザでは別データになります。大事なデータは設定画面からJSONを書き出してください。

## 自動連携について

このMVPでは、銀行・PayPay・モバイルSuicaの自動取得は直接実装していません。CSV取込の口を先に作り、将来は各サービスの公式API、OAuth、またはローカル連携ツールを追加する設計にしています。

## モバイルSuica PDF取込

PDF取込はローカルサーバー経由で動きます。

```powershell
node .\server.mjs
```

`CSV取込` 画面の `モバイルSuica PDF取込` から、モバイルSuicaの `残高ご利用明細` PDFを選択します。金額0の繰越行は取引にせず、チャージは入金、交通利用・物販は支出として取り込みます。PDF内の残高は取込後の照合に使い、初期残高は自動変更しません。
