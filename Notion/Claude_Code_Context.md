- **データ規模**: 約689件のMarkdownファイル。

## 2. 情報アーキテクチャの分析と課題
ユーザーは情報を「属性（タグ）」よりも「場所（フォルダ）」で階層的に管理する傾向があります。
- **主要な4本柱（ドメイン）**:
  1. `Study` (185件): 卒論、TOEIC、簿記、英検など自己研鑽。
  2. `アルバイト` (117件): 単なる労働ではなく、家庭教師（マイクラ活用等）を通じたバリュー提供・ビジネス的視点。

- **課題とネクストアクション**: `Inbox`に約100件の未処理メモが蓄積。今後はフォルダ依存から脱却し、タグ機能（例: `#マネジメント` `#教育`）を活用して分野横断的な「第二の脳」を構築することが推奨されています。

## 3. 構築したシステム①：統合型デイリージャーナル（日次レビュー）
日々の目標、ルーティン、振り返りを構造化したフォーマットを導入しました。
- **目的**: 感情を排除してToDoをこなしつつ、日々の学び（特にビジネス系YouTube等からのインプット）や体調を客観的に記録すること。
- **構造**:
  - `目標`: 長期（年収2000万/資産1億）、中期（資格・卒論）、短期（生活リズム）。
  - `ルーティン`: 朝（日光・整理整頓）、夜（スマホ封印BOX・ストレッチ）。
  - `振り返りカテゴリ`: 「睡眠・体調」「仕事・バリュー提供」「インプット・閃き」「食事・日常」「明日へのアクション」に分割。

## 4. 構築したシステム②：陸上競技の科学的PDCAデータベース
Garminの客観的データと、独自の主観的データを掛け合わせたトラック練習用の管理システムです。
- **データベース設計（Notion）**:
  - `Garmin実測値`: 走行距離、平均/最大心拍数、平均ペース。
  - `主観的評価`: RPE（主観的疲労度1-10）、ケガ・痛みの部位と強度（例: 右足首や膝）。
  - `管理項目`: メニュー種別（Interval, LSD等）、使用シューズ（寿命管理）。
- **ページ本文のPDCAサイクル**:
  - `Plan`: 目的、設定ペース、目標心拍ゾーン、予想RPE。
  - `Do`: 天候、路面、実測データ。
  - `Check`: 予想と結果のギャップ分析、主観・客観のズレの考察。
  - `Action`: （最重要）分析に基づく明日の練習計画の調整・フォーム改善策。

**[Claude Code等 への指示・コンテキスト]**
今後、このシステムに基づいてAIがサポートを行う際は、以下の点に留意してください。

- 分析や計画の立案においては、「感情論」ではなく「客観的データ」と「明確なアクションプラン（Action）」を提示してください。
- 陸上のログや日記の解析を求められた際は、上記の構造的テンプレートに沿ってデータが入力されていることを前提に、不足しているPDCA要素を補完するようなフィードバックを行ってください。
