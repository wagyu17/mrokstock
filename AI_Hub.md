## 🔄 GitHubを中心としたデータ同期・連携ルール

各AIからこの情報源（AI_Hub.md や各種ドキュメント）を参照・編集し、常に最新の状態を保つためのワークフローです。GitHubを「ハブのハブ（中央リポジトリ）」として活用します。

### 1. ローカルAI（Antigravity, Claude Code, Codex）での編集
* **強み**: ローカル環境のファイルに直接アクセス・編集が可能です。
* **運用**: タスクが完了しドキュメントが更新されたら、必ずGitでコミット＆プッシュ（`git commit -m "update" && git push`）を行い、GitHub上のリモートリポジトリを最新に保ちます。

### 2. Web型AI（ChatGPT, Gemini）での編集・参照
* **強み**: スマートフォンや外出先からでもアクセス可能です。
* **Gemini (Web版) の場合**:
  * **Google ドライブ連携**: Desktop全体がGoogleドライブと同期されているため、Web版Gemini（gemini.google.com）から直接ローカルファイルを参照できます。
  * **プロンプト例**: `@Google Drive ドライブ内にある AI_Hub.md というファイルを読んで、現在のプロジェクト一覧を教えて`
* **ChatGPT の場合**:
  * GitHubのリポジトリURL（または生ファイルのURL）をプロンプトで渡し、「現在のAI_Hub.mdの状況を読んでください」と指示してコンテキストを共有します。
* **Webからの編集反映**: Web型AIに出力させたMarkdownは、そのまま**スマートフォンのブラウザからGitHubのWeb画面上で直接ファイル編集（鉛筆マーク）・コミット**を行うと、ローカルに自動でバージョン管理されます。

### 3. 個人ルール（SSOTの徹底）
1. **GitHubを最終的な正とみなす (SSOT: Single Source of Truth)**: どのAIで作業した場合でも、最終的な更新内容は必ずこのローカルフォルダ経由、またはGitHubの画面から直接コミットして同期します。
2. **作業開始時のプル**: PCで作業を始める前は、必ず `git pull` を行い、別のデバイスやAIで更新した内容をローカルに反映させてから開始します。
3. **チャットログに依存しない**: AIとのチャット履歴に重要な情報を残したままにせず、完了したらドキュメントに転記して保存・プッシュします。
