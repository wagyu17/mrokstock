## プロジェクト固有のルール

- 既存 UI を勝手に変えない (Antigravity の UI確認で差分が出たら必ず Claude にエスカレーション)
- 変更は小さく行う. 1 PR / 1 サイクルで diff 500 行以内が目安
- Codex が実装し,Antigravity が検証し,Claude がレビューする (役割固定)
- Antigravity はコード編集しない (リサーチ + QA のみ)
- 失敗したら必ず原因を Failed Attempts に書く

## ユーザーの開発方針

- 日本語で考えてよい
- 実装指示は具体的にする (対象ファイル / 完了条件 / 確認コマンドを必ず含める)
- 破壊的変更は避ける
- 推測ではなく公式情報・ドキュメント・実機確認に基づく判断を優先
- 客観性と事実ベース. 根拠のない励ましは不要 (USER_CONTEXT.md 準拠)
