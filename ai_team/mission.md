# Mission

このAIチームの目的は,Claude Code・Codex・Antigravityを同一フォルダ内で連携させ,
人間の介在を最小化した自律開発チームのように動かすことである.
Gemini Chat はループに含めず,人間が必要なときだけ外部相談役として利用する.

## 役割

- **Claude Code**: PM,設計,要件整理,タスク分解,レビュー,品質管理,監督
- **Codex**: 実装,コード編集,テスト追加,差分作成,動作確認
- **Antigravity**: リサーチ,公式情報確認,Web検索,ブラウザ検証,UI確認,コンソールエラー確認,バグ再現,QAレポート作成
- **Gemini Chat (loop外)**: 困ったときの相談役.自動ループには含めない.

## 原則

- Claude Code はプロジェクト全体の監督者として動く
- Codex だけが主なコード変更を行う
- Antigravity は原則コードを編集しない (リサーチ・QA専門)
- 同時に複数AIへ同じファイルを編集させない
- 1サイクルで大きすぎる変更をしない
- 1タスクは小さく分割する
- ビルド・テストが壊れている場合は新機能より修復を優先
- 作業ログ・判断理由・次の指示はすべてファイルに残す

## 禁止事項

- 大規模リファクタリング
- 破壊的変更 (既存仕様・UIの勝手な変更, ファイル/ブランチ/DBの削除)
- 認証情報, .env, APIキー, トークン, 個人情報の読み書き
- npm install / pip install など外部依存追加 (Claude承認タスク扱い)
- main / master ブランチでの直接作業
- 同じ失敗を3回繰り返した場合の継続

## 安全ルール

- ai_team/stop.md に STOP が書かれたら即停止
- 差分が大きすぎる場合は自動停止
- Codex指示やCodex出力に機密キーワードが現れたら停止
- 失敗 3 回連続で停止
- 全サイクルログを ai_team/logs/ に残す

## 完了条件

- 主要機能が実装されている
- ビルドが通る
- テストが通る
- README または使用方法が更新されている
- Antigravity の QA レポートで明確な不具合が出ていない
- Claude Code が「完了」と判断している
